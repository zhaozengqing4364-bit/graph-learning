# AxonClone 信息架构文档（IA）

## 1. 文档目的

本文档用于定义 AxonClone 的产品信息组织方式，明确系统中的核心对象、页面层级、模块结构、导航关系、数据流向与状态结构，作为产品设计、交互设计、前端架构和后端数据组织的统一依据。

信息架构的目标不是描述视觉样式，而是回答以下问题：

- 系统里有哪些核心信息对象
- 这些对象之间是什么关系
- 页面与页面之间如何组织
- 用户从哪里进入、如何流转、最终沉淀什么
- 哪些内容是主视图，哪些内容是辅助视图
- 如何避免信息过载与图谱型产品常见的迷路问题

---

## 2. 信息架构设计原则

## 2.1 以学习任务为中心，而不是以功能菜单为中心
AxonClone 的结构不应让用户理解为“好多工具”，而应让用户感知为“我在完成一个学习过程”。因此信息架构要围绕学习旅程组织，而不是围绕技术能力组织。

## 2.2 以当前节点为核心，而不是以整个图谱为核心
图谱是全局视图，不应成为默认主视图。默认主视图应始终围绕“我当前最该学什么、最该做什么”。

## 2.3 以主干路径优先，支线探索延后
复杂知识容易发散，因此信息层级必须强制区分主线与支线，避免用户一上来被所有信息同时淹没。

## 2.4 以行动导向组织信息
每一层信息结构都应服务于明确动作，例如：开始学习、继续扩展、进入练习、稍后再学、开始复习、查看表达资产。

## 2.5 以长期沉淀为结构底座
系统不是一次性问答器，所以信息架构必须包含长期资产层：知识资产、表达资产、能力记录、误解记录、待学记录、复习记录。

---

## 3. 顶层信息地图

AxonClone 的顶层信息结构建议拆为六大域：

1. 学习入口域
2. 当前学习域
3. 全局图谱域
4. 表达训练域
5. 成长回顾域
6. 系统配置域

对应的产品主导航建议为：

- 首页
- 学习
- 图谱
- 练习
- 统计
- 设置

其中“学习”是当前节点与当前主题的聚焦学习空间；“图谱”是全局结构空间；“练习”是输出与表达空间；“统计”是成长与复习空间。

---

## 4. 核心信息对象定义

AxonClone 的信息架构首先不是页面架构，而是对象架构。系统核心应围绕以下对象组织。

## 4.1 学习主题 Topic

### 定义
用户当前正在学习的主题容器，可由一个概念、问题、一篇文章或一组资料触发。

### 属性
- topic_id
- title
- source_type
- source_content
- learning_intent
- status
- created_at
- updated_at

### 作用
Topic 是所有节点、路径、练习、总结和统计的上层容器。

### 关系
- 一个 Topic 包含多个 Node
- 一个 Topic 对应多个 Session
- 一个 Topic 对应一张局部图谱
- 一个 Topic 可关联多个 Review Task

## 4.2 知识节点 Node

### 定义
系统中的基础学习单元，用于承载一个概念、机制、方法、问题点或高价值知识对象。

### 属性
- node_id
- name
- summary
- why_it_matters
- applications
- importance
- misconceptions
- examples
- status
- confidence

### 作用
Node 是学习页、图谱、练习、复习和统计的共用基础对象。

### 关系
- Node 属于某个 Topic
- Node 与其他 Node 存在关系边
- Node 可关联多个 Expression Asset
- Node 可关联多个 Ability Record
- Node 可关联多个 Review Item

## 4.3 节点关系 Edge

### 定义
描述两个节点之间学习意义上的连接方式。

### 关系类型建议
- prerequisite
- contrast
- variant
- applies_in
- extends
- misunderstood_as
- evidenced_by

### 属性
- edge_id
- source_node_id
- target_node_id
- relation_type
- reason
- weight
- confidence

### 作用
支撑图谱可视化、路径推荐、卡点回溯和学习顺序。

## 4.4 学习会话 Session

### 定义
用户围绕某一 Topic 在某一时间段内完成的一轮学习过程。

### 属性
- session_id
- topic_id
- start_time
- end_time
- entry_node_id
- visited_nodes
- session_summary
- next_actions

### 作用
Session 是学习闭环、收束总结与复习推荐的基础对象。

## 4.5 学习意图 Learning Intent

### 定义
用户当前希望通过这一轮学习实现什么目标。

### 枚举建议
- 补基础
- 解当前问题
- 建立体系
- 准备表达
- 准备面试
- 写作/分享准备

### 作用
Learning Intent 是路径编排、节点排序和练习生成的重要控制变量。

## 4.6 能力记录 Ability Record

### 定义
用于记录用户在某个节点上的多维能力状态。

### 属性
- user_id
- node_id
- understand_score
- example_score
- contrast_score
- apply_score
- explain_score
- recall_score
- transfer_score
- updated_at

### 作用
构成能力图谱，支撑统计页、复习计划和个性化推荐。

## 4.7 卡点记录 Friction Record

### 定义
记录用户在学习或表达过程中暴露出的阻塞类型。

### 标签建议
- 前置缺失
- 混淆概念
- 缺少例子
- 表达结构弱
- 抽象理解弱
- 调用能力弱
- 回忆失败

### 作用
支撑后续诊断、复习、路径规划与产品个性化体验。

## 4.8 表达资产 Expression Asset

### 定义
沉淀用户在表达训练中形成的可复用表达内容。

### 属性
- asset_id
- node_id
- user_expression
- ai_rewrite
- skeleton
- expression_type
- quality_tags
- created_at

### 作用
形成用户自己的表达素材库和表达风格基础。

## 4.9 误解记录 Misconception Record

### 定义
记录某一节点上的典型错误理解、用户个体误区及纠偏过程。

### 作用
构成误解图谱与个人误解本的底层对象。

## 4.10 复习任务 Review Item

### 定义
由系统根据节点价值、掌握状态与时间间隔生成的复习任务对象。

### 属性
- review_id
- node_id
- review_type
- due_time
- priority
- last_result

### 作用
支撑复习页与回顾页的调度能力。

---

## 5. 对象关系总览

核心对象关系建议如下：

- Topic 是一级容器
- Topic 下包含 Node、Session、Review Item
- Node 通过 Edge 构成知识网络
- Node 关联 Ability Record、Expression Asset、Misconception Record
- Session 连接用户动作轨迹与总结输出
- Learning Intent 作用于 Topic 和 Session
- Friction Record 来自 Session 中的训练反馈，并反哺 Node 与 Review Item

可以理解为：

Topic 管学习范围，
Node 管知识内容，
Edge 管知识结构，
Session 管学习过程，
Ability 管掌握状态，
Expression 管输出资产，
Review 管长期巩固。

## 6. 全局导航架构

## 6.1 一级导航建议

### 1）首页
用于进入学习，查看最近主题、待学堆栈和推荐继续学习内容。

### 2）学习
用于聚焦当前 Topic 和当前 Node，完成知识学习与扩展。

### 3）图谱
用于查看当前 Topic 的全局知识网络和主干/支线结构。

### 4）练习
用于进行表达训练、查看训练记录、管理表达资产。

### 5）统计
用于查看能力变化、卡点分布、复习建议和长期趋势。

### 6）设置
用于调整模型、深度、默认意图、导出等配置项。

## 6.2 导航逻辑原则
- 首页负责进入与恢复
- 学习负责当前节点聚焦
- 图谱负责全局视图
- 练习负责输出训练
- 统计负责长期反馈
- 设置负责系统控制

用户不应频繁跨很多层级才能完成一个动作。核心学习闭环应能在“学习”和“练习”之间完成。

---

## 7. 页面层级架构

## 7.1 首页层级

### 首页包含的主要区块
- 输入区
- 最近学习主题
- 待学堆栈
- 最近练习记录
- 推荐继续学习

### 首页的作用
- 新建 Topic
- 恢复旧 Topic
- 进入待完成节点
- 查看最近成长入口

## 7.2 学习页层级

### 学习页的主对象
- 当前 Topic
- 当前 Node

### 学习页的主要层次
- 顶部：主题信息 / 学习意图 / 当前路径
- 中区主卡：当前节点
- 中区辅助区：前置 / 对比 / 应用 / 误解 / 相关节点
- 底部动作区：继续扩展 / 进入练习 / 稍后再学 / 完成本轮

### 学习页的本质
不是展示所有信息，而是组织“当前最该看的信息 + 当前最该做的动作”。

## 7.3 图谱页层级

### 图谱页主对象
- 当前 Topic 的 Node + Edge 集合

### 图谱页的层次
- 顶部：主题切换 / 视图筛选 / 主干支线切换
- 主区：图谱画布
- 侧区：当前选中节点详情

### 图谱页的本质
提供全局认知，不作为默认主要学习承载页。

## 7.4 练习页层级

### 练习页主对象
- 当前 Node 的训练任务
- 用户作答记录
- AI 反馈
- 表达资产

### 练习页层次
- 顶部：当前练习节点 / 训练类型
- 主区：题目 + 输入区
- 反馈区：点评 + 推荐表达
- 资产区：骨架 / 收藏 / 历史表达

## 7.5 统计页层级

### 统计页主对象
- Ability Record
- Friction Record
- Review Item

### 统计页层次
- 总览：学习主题数、节点数、练习次数
- 能力分布：各维度能力图
- 卡点分布：高频阻塞类型
- 复习建议：待复习节点
- 趋势：近期成长变化

## 7.6 设置页层级

### 设置页主要区块
- 模型配置
- 图谱配置
- 学习配置
- 练习配置
- 数据配置

## 8. 学习域的信息架构

学习域是 AxonClone 的核心区域，必须重点定义其内部结构。

## 8.1 学习域的四层结构

### 第一层：主题层
回答“我现在在学什么主题”。

包含：
- 主题名称
- 来源
- 当前学习意图
- 已覆盖范围

### 第二层：节点层
回答“我当前在学哪个知识点”。

包含：
- 节点名称
- 一句话解释
- 重要性
- 当前掌握状态

### 第三层：关系层
回答“它和其他知识点是什么关系”。

包含：
- 前置
- 对比
- 应用
- 扩展
- 误解

### 第四层：动作层
回答“我下一步要做什么”。

包含：
- 继续学习
- 去练表达
- 暂时跳过
- 查看主线
- 结束本轮

## 8.2 当前节点卡片的结构建议

当前节点卡片建议遵循固定的信息优先级：

1. 节点名称
2. 一句话定义
3. 为什么现在学它
4. 一个直觉例子
5. 常见误解
6. 与相邻概念的对比入口
7. 动作按钮

这样可以保证用户快速进入状态，而不是被信息压住。

---

## 9. 图谱域的信息架构

图谱域必须是辅助性全局视图，而非默认学习页。

## 9.1 图谱域的主要目标
- 看清主干结构
- 定位当前节点
- 查看支线范围
- 理解概念关系
- 决定下一步扩展方向

## 9.2 图谱域的信息组织建议

### 默认视图
只展示主干链与当前高相关节点。

### 进阶视图
允许用户展开更多层级，但必须支持折叠。

### 筛选维度
- 只看主干
- 只看前置
- 只看应用
- 只看误解
- 只看高重要性节点

## 9.3 图谱页与学习页的边界
- 图谱页解决“我在哪”和“整体长什么样”
- 学习页解决“我现在学什么”和“我下一步做什么”

不能让图谱页承载太多学习细节，否则会破坏聚焦体验。

---

## 10. 练习域的信息架构

练习域的核心对象不是“题目”，而是“表达动作”。

## 10.1 训练类型层级
建议至少包含以下训练类型：

- 定义表达
- 举例表达
- 对比表达
- 应用表达
- 面向小白解释
- 压缩表达

## 10.2 练习页内部信息结构

### 任务层
告诉用户现在练什么。

### 输出层
用户输入自己的答案。

### 反馈层
系统点评哪里好、哪里弱。

### 优化层
给出推荐表达与表达骨架。

### 沉淀层
允许保存到表达资产库。

## 10.3 练习信息流原则
- 先让用户说
- 再给反馈
- 再给优化版本
- 再决定是否保存

而不是直接先展示标准答案。

---

## 11. 成长域的信息架构

成长域不是“统计看板”，而是长期能力反馈空间。

## 11.1 主要信息对象
- 能力维度
- 卡点类型
- 高频误解
- 复习优先级
- 表达沉淀情况

## 11.2 建议信息分区

### 总览区
- 学习主题数
- 累计节点数
- 累计练习次数

### 能力区
- 理解能力
- 举例能力
- 对比能力
- 应用能力
- 表达能力
- 回忆能力

### 卡点区
- 前置缺失占比
- 概念混淆占比
- 表达结构问题占比

### 复习区
- 今日待复习
- 本周重点复习主题
- 长期薄弱节点

### 表达区
- 最近新增表达资产
- 高质量表达数量
- 最常用表达骨架

---

## 12. 系统状态架构

除页面信息架构外，还需要定义关键状态结构。

## 12.1 Topic 状态
- 未开始
- 学习中
- 已完成一轮
- 需复习
- 已归档

## 12.2 Node 状态
- 未学习
- 已浏览
- 学习中
- 已练习
- 高混淆
- 待复习
- 已掌握

## 12.3 Session 状态
- 进行中
- 已收束
- 已生成复习任务

## 12.4 Review 状态
- 未开始
- 待完成
- 已完成
- 失败待重练

这些状态将直接影响列表排序、推荐逻辑、视觉标识和后续动作入口。

---

## 13. 信息优先级策略

AxonClone 容易陷入“信息太多”的问题，因此必须定义优先级。

## 13.1 一级优先信息
用户当前立刻需要知道的内容：
- 当前主题
- 当前节点
- 当前节点一句话理解
- 为什么现在学它
- 下一步动作

## 13.2 二级优先信息
用户在学习当前节点时高频需要的辅助内容：
- 前置
- 对比
- 例子
- 误解
- 当前掌握状态

## 13.3 三级优先信息
用户建立全局感时需要的内容：
- 图谱全局结构
- 详细路径
- 历史统计
- 表达资产
- 深层支线

这意味着：默认页面不应先展示三级信息。

---

## 14. 信息流与数据流

## 14.1 用户信息流
输入内容 → 生成 Topic → 抽取 Node → 进入学习页 → 进入练习页 → 生成 Session 总结 → 形成 Ability / Expression / Review 沉淀。

## 14.2 系统数据流
Source Content → AI Parser → Topic / Node / Edge → Session → Ability Update / Friction Update / Expression Asset / Review Item。

## 14.3 关键反哺逻辑
- 练习结果更新能力记录
- 能力记录影响推荐节点
- 卡点记录影响复习优先级
- 表达资产影响后续训练提示
- 误解记录影响节点展示内容

这意味着系统必须是一个循环架构，而不是一次性处理架构。

---

## 15. 关键列表与集合设计

## 15.1 最近学习主题列表
显示用户近期最相关的 Topic。

## 15.2 待学堆栈
保存用户暂时跳过但后续应学习的 Node，并保留上下文来源。

## 15.3 当前主题推荐节点列表
根据意图、主线位置和卡点状态排序。

## 15.4 待复习列表
根据重要性、遗忘风险和能力短板综合排序。

## 15.5 表达资产列表
可按节点、类型、时间和质量标签查看。

---

## 16. 权限与边界（单机版）

当前单机版下，信息架构默认围绕单用户组织，不考虑多人权限系统。

但建议在数据结构上预留：
- user_id
- workspace_id（未来可扩展）
- source_type
- ownership 字段

这样后续更容易扩展多人协作或团队学习空间。

---

## 17. IA 对前端与后端的指导意义

## 17.1 对前端
- 明确页面与模块边界
- 明确哪些对象是一级展示对象
- 明确哪些状态决定 UI 变化
- 明确默认视图与进阶视图的区别

## 17.2 对后端
- 明确对象模型与关系模型
- 明确需要存储的长期资产
- 明确 AI 输出需要落到哪些结构化字段
- 明确哪些对象将参与推荐与复习逻辑

## 17.3 对 AI 层
- 明确输出不只是文本，而是 Topic、Node、Edge、Ability、Friction、Review 等结构对象
- 明确哪些输出需实时生成，哪些可异步补充

---

## 18. 推荐的最终 IA 结论

AxonClone 的信息架构应建立在“Topic—Node—Edge—Session—Ability—Expression—Review”这条主链上。

在用户体验层面，应坚持以下原则：
- 首页负责进入与恢复
- 学习页负责聚焦当前节点
- 图谱页负责全局结构感
- 练习页负责输出训练与沉淀
- 统计页负责长期成长反馈

在系统结构层面，应明确：
- 图谱不是唯一中心，当前节点才是当前体验中心
- 学习不是只产生知识对象，还产生能力对象和表达对象
- 所有页面都应服务于从“知识展开”到“能力形成”的主目标

这套信息架构稳定后，后续无论扩展多模态、任务桥接、多用户协作，底层结构都不会轻易失衡。
